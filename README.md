# pythonGame
attempt to create a game

Небольшой каркас для пошаговой RPG на Python с использованием pygame.

Файлы:
- main.py — точка входа. Создаёт окно pygame и содержит минимальный игровой цикл с переключением ходов (нажатие SPACE).
- run_game.bat — утилита для запуска игры из каталога проекта (удобно использовать в ярлыке Windows).

Требования:
- Python 3.8+ (или совместимая версия)
- pygame

Установка зависимостей:
- Установите pygame: pip install pygame

Как запустить с ярлыка на рабочем столе (Windows):
1. В папке проекта уже есть run_game.bat — он запускает main.py из каталога проекта и пытается использовать pythonw (чтобы не показывать консоль).
2. Создайте ярлык на рабочем столе, указывающий на файл run_game.bat в папке проекта.
   - Правой кнопкой по run_game.bat → Отправить → Рабочий стол (создать ярлык)
   - При желании в свойствах ярлыка укажите "Запуск: Свёрнуто" или поменяйте иконку.
3. Если предпочитаете запускать напрямую через интерпретатор, укажите в ярлыке путь к pythonw.exe и параметром укажите полный путь к main.py; обязательно задайте "Рабочая папка" — папку проекта.

Как запустить в Visual Studio Code:
1. Откройте папку `pythonGame` в VS Code.
2. Убедитесь, что выбран интерпретатор Python (Ctrl+Shift+P → Python: Select Interpreter).
3. Запустите отладку (F5) или выполните `python main.py` в терминале.

Короткие заметки:
- Скрипты используют пути относительно каталога проекта, поэтому запуск через ярлык или bat-файл корректно найдёт config.json и ресурсы.

Дополнительно:
- create_shortcut.py — скрипт для создания ярлыка на рабочем столе с опциями командной строки.
  Примеры:
    - python create_shortcut.py
    - python create_shortcut.py --name "MyGame.lnk" --target run-bat --force
    - python create_shortcut.py --target python --script "C:\path\to\main.py"

Автоустановка pywin32:
- При первом запуске create_shortcut.py скрипт попытается импортировать win32com (pywin32).
- Если модуля нет — скрипт попытается выполнить `python -m pip install pywin32` и затем использовать win32com.
- Если установка или импорт не удастся — скрипт использует VBS-фоллбек для создания ярлыка.

Примечание:
- Для создания ярлыка через win32 может потребоваться права на установку пакетов в текущем окружении; при отказе будет применён VBS.
